Auto-generated by `headroom learn` on 2026-04-07.

## Netlify CLI — General

- `netlify blobs:delete <store> <key>` requires
  the project to be linked
  (`netlify link --name projectinsomnia`).
  Without linking it fails with
  'You don't appear to be in a folder
  that is linked to a project'.
- `netlify logs:function <name>` streams and does
  not return — use
  `timeout 8 netlify logs:function <name>`
  or avoid it. The `--timeout`, `--tail`, and
  `--level` options do not exist.
- `netlify functions:log` is not a valid command;
  use `netlify logs:function`.

## Netlify CLI — API Methods

- `netlify api createSiteDomain` and
  `netlify api listSiteAliases` are not valid
  API methods. Use `netlify api updateSite` with
  `body.domain_aliases` or `body.custom_domain`
  to manage domains.
- `netlify api provisionSiteTLSCertificate`
  returns 422 Unprocessable Entity if DNS has
  not propagated yet; retry after DNS is
  confirmed.

## Git Workflow

- Never commit directly to `main`; a pre-commit
  hook blocks it. Always
  `git checkout -b claude/<feature>-<date>`
  first.
- Branch naming convention:
  `claude/<type>-<description>-<date>` (e.g.,
  `claude/fix-instagram-shortcode-20260313`).
- Merging requires human-granted merge-lock
  authorization via
  `~/.claude/hooks/merge-lock.sh authorize`
  — Claude cannot run this command
  (hook blocks it).

## GitHub CLI

- `gh pr merge <N> --squash` fails with a hook
  error; always use
  `gh pr merge <N> --squash --delete-branch`.
- `gh -C <path> pr list` fails (unknown flag);
  run `gh pr list` from the repo directory or
  use `gh pr view <N> --repo smartwatermelon/projectinsomnia`.

## PR and CI Checks

- `gh pr checks <N>` returns exit code 8
  (non-zero) when checks are pending, and
  GraphQL errors when the PAT lacks
  `statusCheckRollup` access. This is normal;
  use `bash ~/.claude/scripts/post-push-status.sh <N>`
  for polling CI status instead.
- Wait at least 60-90 seconds between CI status
  polls; 30-second intervals are usually too
  fast.

## Pre-commit Hook

- The pre-commit hook runs markdownlint on `.md`
  files and will fail commits with lint errors.
  Run `npx markdownlint-cli2 <file>` before
  committing to catch issues early.
- If markdownlint fails, the second `git commit`
  attempt (after fixing lint) usually succeeds
  — the hook re-runs and passes.

## MCP Tools

  `mcp__headroom__headroom_retrieve`,
  NOT `headroom_retrieve`. Using the bare name
  fails with 'No such tool available' every
  time.

## File Paths

- Spec file is at `docs/SPEC.md` (uppercase),
  not `docs/spec.md`. Backlog is at
  `docs/BACKLOG.md`. Content config is at
  `src/content.config.ts`, not
  `src/content/config.ts`.
- Blog posts live in `src/content/blog/`; images
  in `public/images/blog/` (served as
  `/images/blog/`).

## Build Commands

- Use `npm run build` or `npx astro build` or
  `pnpm build` (project uses pnpm in some
  sessions); check `package.json` scripts if
  unsure. `npx tsc --noEmit` for type-checking.
- `npx astro check` also works for type/SSR
  validation.

## Blog Post Voice

- Before drafting or editing any post in
  `src/content/blog/`, read `docs/VOICE.md`
  and apply it.

  vocab, numbered-list lead-ins, italics,
